Installers/user’s Manual

Shaula 720



INTRODUCTION:


Shaula 720 is one of the advanced security and smart Home automation system from Astrolabe tech. In order to use it properly and get most out of it, the Installer should follow this guide. This guide describes Programming and setup of the system. It will help the reader to understand the commands and their structures with help of examples.

USERS:

Shaula 720 supports three types of Users, which we will explain in detail. These are:


  • Super User (SU)

  • Master User (MU)

  • Normal User (NU)


Each of these users has different roles, functions and access levels. Furthermore the system supports specific number of these users based on the firmware of the board.


Super user:

There is only one super user in every firmware. Super User has the highest access level. A super user can only be added when the system is in factory default status or if a soft reset has been applied, there are no other users in both these conditions. Until a super user has been added Shuala 720 will function with minimum functionality. It will work on remote control and keypad (if installed), will make siren if an alarm is activated, but there will be no calls. Furthermore all the zones will be set as General alarms and will follow Away mode in case of default state,

A super user can be deleted only by default and soft reset. While adding a Super User a password is also required to be chosen, this will be used when using the TCP Remote app.


Master users:

A minimum of 8 Master Users are supported by the system. Some firmware can support more, exact details can be found in the firmware details section. Master users can be added once a Super user has been added to the system. Each master user is given its Access level by the Super User while adding. Access levels and permissions details will come ahead in the details of command for adding a master user. A master user can be deleted by a command as well. While adding a Master User a password is also required to be chosen, this will be used when using the TCP Remote app. This password can be changed later by change password command.


Normal users:

Normal users can vary from none to 8. Normal users have no access to the system through any App. They have no passwords. They can only Open and close the Entry gates via Call or SMS.

ZONES

Shaula 720 supports 16 Physical zones and 4 Virtual Zones. Each zone can be given a name not more than 15 chars; this makes a better idea of the zone when an alert is received. Each zone has some parameters listed below:

  • Name

  • Type

  • Mode

  • Enable/Disable Zone


Name:

Each zone has a name, that helps identify the area for the zone. Name can be max 15 characters long.

Type:

A zone can be one of the following types.

  • General Alarm : Makes an alarm if armed and the zone follows the current mode of System and the zone is enabled.

  • Panic : Makes an alarm if armed and zone is enabled. (Panic Alarms don’t make sound)

  • Fire : makes a Fire alarm if alarm is produced from this zone.

  • 24 Hours : it’s a general alarm that doesn’t follow any zone and works even if disarmed.





Mode

Shaula 720 can be armed in two modes. Stay and Away. Each zone can be configured to follow Stay Mode, Away mode or both if its type is to General Alarm. When the system is armed in Away Mode, siren makes three squawks. Two when disarmed and one if armed in stay mode. Using remote if same button is pressed again the system will ignore it if the last press was within 10 seconds.

Enable/disable Zone

A zone can be disabled or made enabled. A disabled zone is completely ignored by the system even if some sensors are connected.
